# SMB (445)

## Introduction to SMB&#x20;

SMB (Server Message Block) is Microsoft's **Application Layer network protocol** that allows computers to **share resources over a network**.

Those resources include:

* Files
* Directories
* Printers
* Named Pipes
* Communication Channels
* Some Remote Management Functions

{% hint style="info" %}
_SMB doesn't only share folders. But in reality, SMB is Microsoft's primary network file system and communication protocol. Many other windows protocols like MSRPC, DCOM, remote registry and Service Control Manager actually travel through SMB._&#x20;
{% endhint %}

### SMB is Stateful&#x20;

Unlike HTTP, SMB keeps a persistent session.

```
Client
   │
TCP Connection
   │
Authentication
   │
Session Created
   │
Open Files
Read
Write
Rename
Delete
Close
Disconnect
```

Everything happens inside one authenticated session.

### SMB Ports

SMB has historically used **two TCP ports**:

* **TCP 139** (NetBIOS Session Service)
* **TCP 445** (Direct SMB)

{% hint style="info" %}
_Although both can provide SMB access, **modern Windows primarily uses TCP 445**._
{% endhint %}

### How Windows handles a share request&#x20;

When a client requests:

```
\\SERVER\Finance
```

Windows roughly performs the following steps:

{% code overflow="wrap" %}
```
Client
    │
    ▼
Connect to SMB Server
    │
Authenticate User
    │
Locate Share "Finance"
    │
Check Share Permissions
    │
Check NTFS Permissions
    │
Grant or Deny Access
```
{% endcode %}

Notice that **share permissions are only one part of the access check**. NTFS permissions are also evaluated.

### Connecting to IPC$

Suppose you execute:

```
net use \\192.168.16.132\IPC$ /user:administrator Password123
```

Windows does **not** mount a folder.

Instead, it establishes an **authenticated SMB session** that can later be used to access named pipes and remote services.

### SMB Signing&#x20;

SMB Signing is a security feature that adds a **cryptographic signature** to every SMB message.

The receiving system verifies this signature before accepting the message.

SMB Signing provides:

* **Integrity** ✅ — Detects if SMB packets were modified.
* **Authenticity** ✅ — Confirms the packet came from the authenticated SMB session.

#### Required V/s Enabled

* **Enabled** ⇒ The server is capable of using SMB Signing. if the client does not negotiate signing, communication may still proceed unsigned (depending on configuration and protocol version).
* **Required** ⇒ Every SMB packet **must** be signed. Unsigned SMB sessions are not permitted.

### SMB Encryption&#x20;

SMB Encryption encrypts the **entire SMB payload** before it is transmitted across the network.

SMB Encryption provides:

* **Confidentiality** ✅ — Prevents others from reading SMB data.
* **Integrity** ✅ — Detects unauthorized modification as part of the encrypted communication.
* **Authentication** ✅ — Used within an authenticated SMB session.

It is available in SMBv3.&#x20;

### SMB Session (Authentication)

An SMB Session is the authenticated communication channel between your computer and the SMB server that is made post successful authentication to the SMB Server.&#x20;

Everything happens over the same authenticated session.

### SMB Mapping&#x20;

A mapping is simply Windows keeping track of a connected share.&#x20;

## SMB Lab

### Create a share&#x20;

{% hint style="info" %}
**Note**

* _**By default, SMB shares require authentication.** Anonymous (null session) access is **not** allowed on modern Windows systems._
* _**Authenticated users can access a share only if the Share Permissions allow them.**_
* _**By default, a newly created SMB share often grants the****&#x20;****`Everyone`****&#x20;****group the****&#x20;****`Read`****&#x20;****Share Permission** (depending on how the share was created and Windows version/configuration)._
* _**The****&#x20;****`Everyone`****&#x20;****group includes authenticated users, but it is not the same as "all users on the network."** Modern Windows excludes anonymous users from the `Everyone` group._
* _**After Share Permissions are checked, NTFS permissions are evaluated.**_
* _**The effective permission is the most restrictive combination of Share Permissions and NTFS permissions.**_
* _By default, Windows enables the File Sharing service (SMB) across all active network interfaces (like Wi-Fi and Ethernet)._
{% endhint %}
