# RDP (3389)

**Remote Desktop Protocol (RDP)** is a **proprietary remote access protocol developed by Microsoft** that allows a user to **graphically connect to and control another Windows computer over a network**.

## Security Stack&#x20;

Instead of thinking of RDP as a single protocol, think of it as multiple security layers working together.

```
          User
            │
            ▼
+-------------------------------+
| Remote Desktop Protocol (RDP) |
+-------------------------------+
| Network Level Authentication  |
| (NLA)                         |
+-------------------------------+
| CredSSP                       |
+-------------------------------+
| TLS                           |
+-------------------------------+
| TCP                           |
+-------------------------------+
| IP                            |
+-------------------------------+
```

### TLS (Transport Layer Security)

**TLS (Transport Layer Security)** encrypts communication between the RDP client and the RDP server.

Its job is simple:

> **Prevent attackers from reading or modifying data while it travels across the network.**

With **TLS** everything is encrypted and anyone capturing the packets only sees encrypted data.&#x20;

### NLA (Network level Authentication)

NLA offers several security and operational benefits:

* Reduces unnecessary resource usage by rejecting invalid users before creating a desktop.
* Lowers exposure to some attacks that target the pre-authentication desktop/session creation process.
* Ensures only authenticated users reach the interactive desktop stage.

For penetration testers, NLA means you typically cannot interact with the Windows logon screen over RDP unless valid credentials are supplied first.

## RDP Protocol Versions&#x20;

Microsoft has revised the RDP protocol multiple times, mostly tied to Windows/Windows Server releases. As an administrator, the RDP version determines what security and performance features are available.

## Enabling and Configuring RDP&#x20;

### Enable RDP (GUI)

#### Way - 1 Using Settings&#x20;

{% code overflow="wrap" %}
```
Settings → System → Remote Desktop → toggle On
```
{% endcode %}

<figure><img src="../../../.gitbook/assets/image (1498).png" alt=""><figcaption></figcaption></figure>

#### Way - 2 Using This PC&#x20;

{% code overflow="wrap" %}
```
This PC → Properties → Remote Settings → Allow remote connections to this computer
```
{% endcode %}

<figure><img src="../../../.gitbook/assets/image (1499).png" alt=""><figcaption></figcaption></figure>

#### Way - 3 Using Registry&#x20;

The master switch is the `fDenyTSConnections` value:

```
HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server
    fDenyTSConnections = 0   (Enabled)
    fDenyTSConnections = 1   (Disabled)
```

**Via PowerShell**&#x20;

{% code overflow="wrap" %}
```
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' `
  -Name "fDenyTSConnections" -Value 0
```
{% endcode %}

<figure><img src="../../../.gitbook/assets/image (1500).png" alt=""><figcaption></figcaption></figure>

#### Via CMD&#x20;

{% code overflow="wrap" %}
```
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server" /v fDenyTSConnections /t REG_DWORD /d 0 /f
```
{% endcode %}

<figure><img src="../../../.gitbook/assets/image (1503).png" alt=""><figcaption></figcaption></figure>

## Who can Log In via RDP ?&#x20;

By default, Windows allows members of certain local groups to connect.

The two most important are:

```
Administrators
```

and

```
Remote Desktop Users
```

### 1. Administrators Group

Local administrators automatically have permission to use RDP (unless explicitly restricted by policy).

Example:

```
AdministratorJohn (Local Admin)ITAdmin
```

These users usually don't need to be added to any additional group.

### 2. Remote Desktop Users Group

Windows provides a built-in local group named:

```
Remote Desktop Users
```

This allows **non-administrative users** to connect via RDP.

Example:

```
Local Groups

Administrators

Remote Desktop Users
     │
     ├── Alice
     ├── Bob
     └── HelpDesk
```

These users:

* Can establish an RDP session.
* Do **not** automatically receive administrative privileges.
* Operate with the permissions assigned to their own account.

## Adding a User&#x20;

### Using PowerShell&#x20;

{% code overflow="wrap" %}
```powershell
Add-LocalGroupMember -Group "Remote Desktop Users" -Member alice
```
{% endcode %}
