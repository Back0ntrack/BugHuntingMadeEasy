# NFS (2049)

NFS (Network File System) is a distributed file sharing protocol that allows a computer to access files and directories stored on another computer over a network as if they were local.

Instead of copying files between systems, NFS allows multiple systems to directly access the same shared storage.

## NFS Components&#x20;

### NFS Server&#x20;

Hosts the shared directories.

Responsible for:

* exporting folders
* checking permissions
* serving file requests

### NFS Client&#x20;

Requests access to exported directories.

Mounted directories appear as local folders.

### Shared Directory&#x20;

A directory intentionally shared by the server.

Example:

```
/srv/share
```

### Mount Point&#x20;

The local directory where the client accesses the remote share.

Example:

```
/mnt/nfs
```

### Export Options&#x20;

<table><thead><tr><th width="190">Option</th><th>Meaning</th></tr></thead><tbody><tr><td><code>ro</code></td><td>Read-only export</td></tr><tr><td><code>rw</code></td><td>Read-write export</td></tr><tr><td><code>sync</code></td><td>Server replies only after changes are written to disk</td></tr><tr><td><code>async</code></td><td>Server replies before changes are written to disk (faster, unsafe)</td></tr><tr><td><code>subtree_check</code></td><td>It verifies that every accessed file is still inside the exported directory.</td></tr><tr><td><code>no_subtree_check</code></td><td>Disables subtree checking (recommended, avoids extra checks when exporting a subdirectory)</td></tr></tbody></table>

### Squash Options&#x20;

<table><thead><tr><th width="186.79998779296875">Option</th><th width="470.79998779296875">Meaning</th></tr></thead><tbody><tr><td><code>root_squash</code></td><td><strong>Only remote root (UID 0)</strong> → anonymous user (<code>nobody</code>) (default, safe)</td></tr><tr><td><code>no_root_squash</code></td><td>Remote root keeps root privileges on share (dangerous)</td></tr><tr><td><code>all_squash</code></td><td>Maps ALL remote users (including root) to <code>nobody</code></td></tr><tr><td><code>anonuid</code></td><td>Sets the UID that squashed users are mapped to</td></tr><tr><td><code>anongid</code></td><td>Sets the GID that squashed users are mapped to</td></tr></tbody></table>

**Sample /etc/exports Line**

```bash
/srv/nfs/share 192.168.16.0/24(rw,sync,no_subtree_check,no_root_squash)
```

**anonuid / anongid Example**

```bash
/srv/nfs/share 192.168.16.0/24(rw,sync,all_squash,anonuid=1000,anongid=1000)
# Remote users will be seen as user with UID and GID 1000. 
```

`no_root_squash` + `rw` together = client root can read/write as root on the share. Classic NFS privesc combo.

### Writable Share&#x20;

For making a share writable add this permission to the exported folder:&#x20;

{% code overflow="wrap" %}
```
sudo chmod 777 <folder>
```
